# 《狂人日记》代表作品详情样板设计说明

- 项目：横眉·鲁迅文化数字展馆
- 设计日期：2026-07-21
- 适用模块：代表作品列表、通用内容详情、本地备用数据、Supabase 内容同步
- 样板内容：鲁迅《狂人日记》
- 状态：已完成需求确认，等待用户复核

## 1. 设计目标

在不改变现有 Supabase 数据表结构、详情页 URL、搜索逻辑和收藏逻辑的前提下，将《狂人日记》制作成完整的代表作品详情样板。

样板应同时验证以下能力：

1. 1400～1800 字原创整理正文的网页阅读体验。
2. 轻量标记正文的安全解析与结构化展示。
3. 作品封面、档案信息、图片来源及授权说明的展示。
4. 1～2 段经版本核对的作品原文短引。
5. 每部作品独立的 4～6 条参考资料及正文编号跳转。
6. Supabase 云端数据与 `assets/data/works.json` 本地备用数据一致。
7. 桌面端、平板端和 375×812 手机端均可正常阅读。
8. 既有作品赏析、历史影像和收藏功能不发生回归。

本样板通过验收后，再沿用同一结构批量完善《阿Q正传》《故乡》《祝福》《朝花夕拾》《野草》。


当前架构特点：

- `content-list.js` 负责卡片、详情入口和收藏按钮初始化。
- `detail.html` 同时服务 `works`、`articles`、`gallery` 三种内容类型。
- `detail.js` 根据 URL 参数读取内容并渲染元数据、图片和正文。
- `content-service.js` 优先读取 Supabase，失败时读取本地 JSON。
- 当前 `detail.js` 仅按空行把 `body` 拆成普通段落。
- 当前 `works.json` 已包含六部作品，且《狂人日记》的稳定编号为 `kuangren-riji`。

## 4. 《狂人日记》正文结构

《狂人日记》正文控制在约 1400～1800 字，采用原创整理与原创赏析，作品原文只作为辅助材料。

章节顺序固定为：

```text
作品导读
创作与发表背景
内容概述
主要人物与叙述视角
核心主题
艺术特色
文学史影响
参考资料
```

内容要求：

- “作品导读”说明作品的基本价值和阅读入口。
- “创作与发表背景”说明写作时代、首次发表与新文学语境。
- “内容概述”只概括叙事进程，不大段复述原作。
- “主要人物与叙述视角”重点解释狂人、旁观者及日记叙述的关系。
- “核心主题”分析“吃人”、觉醒者孤独和“救救孩子”等层次。
- “艺术特色”分析日记体、不可靠叙述、象征、反讽和语言特点。
- “文学史影响”说明作品在现代文学史中的位置。
- 每部作品独立列出 4～6 条参考资料。
- 加入 1～2 段经版本核对的原文短引，并标明《狂人日记》。

## 5. `body` 轻量标记规范

不修改 `contents` 表结构，正文继续保存在一个 `body` 字段中。

支持的标记只有：

```text
## 章节标题

普通段落。

> 原文短引
> ——《狂人日记》

- 要点一
- 要点二

正文中的资料编号[1][2]

## 参考资料

[1] 资料条目
[2] 资料条目
```

### 5.1 解析规则

- 以 `## ` 开头的独立行生成二级章节标题。
- 连续的 `> ` 行合并为一个引用框。
- 连续的 `- ` 行合并为一个无序列表。
- 普通连续文本按空行分段。
- 正文中的 `[1]`、`[2]` 等生成可点击来源编号。
- `## 参考资料` 后，以 `[数字]` 开头的条目生成参考资料列表。
- 参考资料获得稳定锚点，例如 `reference-1`。
- 正文编号获得唯一锚点，例如 `citation-1-1`。
- 参考资料条目提供“返回正文”链接，返回该编号第一次出现的位置。
- 仅自动识别 `https://` URL；外部链接添加 `target="_blank"` 与 `rel="noopener noreferrer"`。
- 未识别或格式错误的标记按普通文本显示，不静默丢失。
- 没有轻量标记的旧正文继续按普通段落渲染。

### 5.2 安全要求

解析器必须使用 DOM API、`createTextNode` 和 `textContent` 创建内容，不把正文直接写入 `innerHTML`。

正文中出现的 HTML 标签应作为普通文字显示，不允许执行脚本、事件属性或嵌入式内容。

## 6. 数据结构

继续使用现有字段：

```text
title
summary
body
image_path
metadata
```

《狂人日记》的稳定定位条件为：

```text
content_type = 'works'
slug = 'kuangren-riji'
```

### 6.1 `metadata` 扩展字段

保留现有字段：

```text
year
genre
collection
first_published
keywords
```

新增可选字段：

```text
creation_place
image_type
image_caption
image_creator
image_date
image_source
image_license
image_alt
```

字段含义：

- `creation_place`：创作地点或主要写作地点。
- `image_type`：`historical` 或 `original`。
- `image_caption`：封面或书影说明。
- `image_creator`：原作者、设计者或资料制作机构。
- `image_date`：图片、书影或本站原创封面的年代。
- `image_source`：原始来源页面或“本站原创”。
- `image_license`：公共领域、CC 许可或本站原创说明。
- `image_alt`：无障碍替代文字。

这些字段均为可选，不影响现有内容。

## 7. 封面策略

采用混合封面方案：

1. 优先核验授权明确、清晰度合适的早期书影。
2. 历史书影不满足清晰度、授权或构图要求时，使用本站原创视觉封面。
3. 本站原创封面采用统一民国风格：
   - 米白纸张底色；
   - 暗红细边框；
   - 竖排或集中式书名；
   - 小型印章元素；
   - 不仿冒具体出版社版本。
4. 页面明确标注“历史书影”或“本站原创视觉封面”。
5. 图片文件保存在：

```text
assets/images/works/
```

6. 图片授权登记保存在：

```text
docs/鲁迅代表作品图片来源与授权登记.md
```

## 8. 详情页视觉结构

### 8.1 桌面端

作品类型详情采用两列档案首屏：

```text
┌──────────────┬──────────────────────────────┐
│   3:4 封面   │  代表作品 · 小说             │
│   档案信息   │  狂人日记                    │
│   授权说明   │  摘要、标签、收藏与返回入口   │
└──────────────┴──────────────────────────────┘
┌─────────────────────────────────────────────┐
│                长文阅读区                    │
└─────────────────────────────────────────────┘
```

正文阅读宽度控制在约 720～780px，并在宽屏中居中，避免单行过长。

作品首屏可在不大规模重写 HTML 的情况下实现：

- `detail.js` 为文章根节点添加 `detail-article--works` 等类型类名。
- 作品类型使用 CSS 网格重新安排现有 `detail-heading`、`detail-sidebar` 和 `detail-content`。
- 赏析与影像继续使用现有布局。
- `detail.html` 只增加必要的图片说明容器，不重建页面。

### 8.2 封面

- 封面目标比例为 3:4。
- 历史书影使用 `object-fit: contain`，避免裁掉书名和边缘。
- 作品类型无图或图片加载失败时，显示统一原创文字封面占位。
- 影像类型仍保留原有大图展示逻辑。

### 8.3 章节与正文

- 章节标题使用衬线字体、暗红装饰线和清晰层级。
- 正文字号桌面端约 17px，行高约 2。
- 原文短引使用浅米色背景和暗红左边线。
- 引文出处右对齐。
- 要点列表只承担提炼作用，核心分析仍使用完整段落。
- 参考资料区在正文末尾独立分隔。

### 8.4 手机端

在 375×812 下：

- 首屏变为单列。
- 封面居中，宽度约为视口的 55%～65%。
- 作品标题、摘要、标签和按钮位于封面下方。
- 按钮空间不足时自动换行或上下排列。
- 正文左右留白约 18～22px。
- 引用框缩小内边距。
- 长资料名称和 URL 自动换行。
- 不出现横向滚动条。

## 9. 三种内容类型的隔离

`detail.html` 同时服务三种类型，渲染必须按类型隔离。

### 9.1 代表作品 `works`

- 启用作品封面档案首屏。
- 启用完整轻量标记正文。
- 展示作品专属元数据及图片授权说明。

### 9.2 作品赏析 `articles`

- 保留作者、关联作品、赏析角度和阅读时间。
- 可以解析轻量标记，但不套用作品封面布局。
- 旧数据仍能正常显示。

### 9.3 历史影像 `gallery`

- 保留大图、时间、地点、分类和授权信息。
- 不把历史影像裁成 3:4 作品封面。
- 图片加载失败时继续显示影像占位提示。

## 11. Supabase 同步

新增独立迁移：

```text
supabase/migrations/011_update_kuangren_riji_sample.sql
```

迁移只更新目标记录：

```sql
where content_type = 'works'
  and slug = 'kuangren-riji'
```

只更新：

```text
summary
body
image_path
metadata
updated_at
```

迁移必须具备以下保护：

- 不插入第二条同 slug 内容。
- 不修改其他作品、赏析或影像。
- 不修改状态、排序和主键。
- 使用 JSONB 合并方式保留未涉及的元数据字段。
- 更新后可通过查询确认仅一行受影响。

本地 `assets/data/works.json` 与迁移内容保持一致。

## 12. 错误处理

### 12.1 正文为空

显示：

```text
相关内容正在整理中。
```

### 12.2 标记格式错误

按普通文本显示，并可在控制台输出非阻断警告。

### 12.3 引用编号无对应资料

编号仍显示为普通文字，不生成失效链接，并输出非阻断警告。

### 12.4 资料存在但正文未引用

资料条目仍正常显示，不阻断页面。

### 12.5 封面为空或加载失败

显示统一文字封面占位，并继续显示正文、元数据和收藏按钮。

### 12.6 Supabase 不可用

回退到 `works.json`，页面显示“数据来源：本地备用”。
